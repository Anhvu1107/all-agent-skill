# Platform Routing

Use this before diving into the larger mobile design docs.

## Pick The Platform Shape

- iOS-only: read `platform-ios.md` early and keep interaction language native-first.
- Android-only: read `platform-android.md` early and keep Material and device variance in mind.
- Cross-platform: read both platform files, then resolve differences intentionally instead of defaulting to one platform's patterns.

## Load The Right Root Guides

- navigation question: `mobile-navigation.md`
- performance or perceived speed question: `mobile-performance.md`
- gesture or touch ergonomics question: `touch-psychology.md`
- backend or offline sync question: `mobile-backend.md`
- testing question: `mobile-testing.md`
