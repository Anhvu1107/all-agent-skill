# Template Vs Scratch

Choose the deck path before touching slides.

## Use Template Editing When

- an existing deck, brand template, or visual language must be preserved
- speaker notes, masters, or layout geometry already exist
- the user wants updates rather than a reinvention

Read `editing.md` and keep the existing structure intact.

## Use Scratch Generation When

- no useful template exists
- the deck is content-first and can be rebuilt cleanly
- the ask is primarily about creating a new presentation artifact

Read `pptxgenjs.md` and decide theme, slide system, and content structure up front.
